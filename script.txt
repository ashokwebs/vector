KNOWLEDGE BASE
(Visual: Presenter clicks on 'Knowledge Base' in the sidebar. Screen shows an upload area and a list of indexed documents.)

Presenter:
"Of course, for these agents to be truly effective, they need context. They need to know our specific company data. That's where the Knowledge Base comes in. 

(Visual: Presenter drags and drops a PDF into the glowing upload zone.)

I'm going to drag and drop our 'Q2 Financial Report.pdf' into this upload zone. When I release it, you'll see a progress bar. Behind the scenes, the system is parsing the document, breaking it down into chunks, generating embeddings, and storing them in our secure PostgreSQL vector database.

We've engineered this heavily for stability. We've resolved complex embedding dimension mismatches that commonly plague RAG (Retrieval-Augmented Generation) systems. Furthermore, we've built in robust fallback mechanisms. If an API key for the primary embedding model is missing or hits a rate limit, the system gracefully falls back to a secondary local model without crashing the UI. This guarantees uninterrupted operations during critical planning sessions."
